# HEARTBEAT.md

On heartbeat:

1. Review `TASKS.md`.
2. Review `MEMORY.md` only if needed for context.
3. Review today's daily memory file only if needed for context.
4. Check for any of the following:
   - a high-priority item in `Next` that appears stalled
   - any item in `Waiting` that likely needs follow-up
   - any item in `Scheduled` that is due or close to due
   - any obvious setup gap in the current operating core
5. If something genuinely needs Jacob's attention, send a short, direct alert.
6. If nothing needs attention, reply exactly `HEARTBEAT_OK`.

Constraints:
- Stay quiet unless there is something actionable or important.
- Do not invent work from old chats if it is not reflected in current files.
- Keep heartbeat work lightweight and low-token.
- Prefer identifying drift, blockage, deadlines, or missing follow-up.
